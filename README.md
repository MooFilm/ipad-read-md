# ReadShelf Personal

เว็บแอปอ่านไฟล์ Markdown สำหรับ iPad ที่เปิดผ่าน GitHub Pages ได้ และสามารถเลือกเก็บหนังสือในเครื่อง หรืออัปขึ้น GitHub เพื่อซิงก์ไฟล์ข้ามอุปกรณ์

## แนวคิดของเวอร์ชันนี้

- GitHub เก็บตัวโปรแกรม และไฟล์หนังสือไว้ใน `public/docs` เมื่อเปิดโหมด GitHub Cloud
- ข้อมูลการอ่าน (progress, bookmark) ยังเก็บในเครื่องด้วย `IndexedDB`
- ถ้าเบราว์เซอร์ไม่รองรับหรือมีปัญหา ระบบจะ fallback ไป `localStorage`
- เหมาะกับการเปิดจาก Safari บน iPad แล้วกด `Add to Home Screen`

## เริ่มใช้งานในเครื่อง

```bash
npm install
npm run dev
```

ถ้าต้องการทดสอบจากอุปกรณ์อื่นในวงเดียวกันชั่วคราว:

```bash
npm run dev -- --host
```

## Deploy ขึ้น GitHub Pages

โปรเจกต์นี้ deploy อัตโนมัติเมื่อ push ขึ้น branch `main`

1. push โค้ดขึ้น GitHub
2. ไปที่ `Settings > Pages`
3. เลือก `Source = GitHub Actions`
4. รอ workflow ทำงานเสร็จ
5. เปิดลิงก์ GitHub Pages บน iPad

ตัวอย่าง URL:

```text
https://YOUR-USERNAME.github.io/YOUR-REPOSITORY/
```

## ใช้งานแบบไม่ต้องใช้ Git CLI (แนะนำ)

ถ้าต้องการอัปโหลดไฟล์จากคอมแล้วเปิดอ่านบน iPad ได้ทันที โดยไม่ต้องใช้ git CLI:

1. ไปที่ `public/readshelf.config.json` แล้วใส่ค่า repo (เช่น `owner/repo`)
2. ตั้ง `syncMode` เป็น `"auto"` เพื่อให้ซิงก์อัตโนมัติเมื่อเปิดแอปหรือกลับมาใช้งาน
3. push ไฟล์ขึ้น GitHub (ทำผ่านเว็บ GitHub ได้)
4. อัปโหลดไฟล์ .md ผ่านหน้าเว็บ GitHub ไปที่โฟลเดอร์ `public/docs`
5. เปิด GitHub Pages บน iPad หรือกลับมาใช้งาน → ระบบจะซิงก์ให้อัตโนมัติแบบอ่านอย่างเดียว

> แนะนำให้ใช้ repo แบบ public เพื่อให้ซิงก์ได้โดยไม่ต้องใส่ token

## วิธีใช้งานจริง

1. เปิดเว็บแอปบน iPad
2. กด `เพิ่มหนังสือ`
3. เลือกไฟล์ `.md`, `.markdown` หรือ `.txt` จากเครื่อง
4. ถ้าต้องการจัดหมวด ให้กด `สร้างโฟลเดอร์` ก่อนแล้วค่อย import
5. ถ้าตั้งค่า GitHub Cloud ไว้ ระบบจะอัปโหลดไฟล์ขึ้น `public/docs` ให้ทันที
6. เปิดอ่าน ระบบจะจำ progress และ bookmark ให้อัตโนมัติในเครื่องนั้น

## GitHub Cloud (อัปโหลด + ซิงก์ไฟล์)

เมื่ออยากเปิดอ่านจากหลายอุปกรณ์:

1. ถ้า repo เป็น public ให้ตั้งค่าใน `public/readshelf.config.json` เพื่อซิงก์แบบอ่านอย่างเดียว
2. ถ้าต้องการอัปโหลดจาก iPad ให้กด `ตั้งค่า GitHub`
3. ใส่ `owner/repo` และ Personal Access Token (ต้องมีสิทธิ์เขียน repo)
4. ระบบจะใช้โฟลเดอร์ `public/docs` เป็นที่เก็บไฟล์
5. ถ้าเลือกโหมดกดซิงก์เอง ให้กด `ซิงก์ GitHub` เพื่อดึงไฟล์ล่าสุดจาก repo (โหมดอัตโนมัติจะซิงก์เมื่อเปิดแอปหรือกลับมาใช้งาน)

กติกาชื่อไฟล์และโฟลเดอร์:

- ไฟล์ต้องมีนามสกุล `.md`, `.markdown`, `.txt`
- หลีกเลี่ยงชื่อซ้ำในโฟลเดอร์เดียวกัน (ไม่แยกตัวพิมพ์เล็ก-ใหญ่)
- ไม่ควรมี `/` หรือ `\` ในชื่อไฟล์

## ความเป็นส่วนตัว

- คนอื่นที่เปิดลิงก์เดียวกันจะไม่เห็น progress หรือ bookmark ของคุณ
- ถ้าใช้ GitHub Cloud ไฟล์ที่อยู่ใน repo จะถูกแชร์ตามสิทธิ์ของ repo นั้น
- อย่านำไฟล์ส่วนตัวไปไว้ใน `public/docs` ถ้าไม่ต้องการให้เป็นสาธารณะ

## การสำรองข้อมูล

แอปมีปุ่ม `ส่งออกแบ็กอัป` และ `นำเข้าแบ็กอัป`

ควรสำรองข้อมูลก่อน:

- ล้างข้อมูลเบราว์เซอร์
- เปลี่ยนเครื่อง
- ย้ายจาก Safari ไปเบราว์เซอร์อื่น
- ทดลองรีเซ็ตแอป

## จุดเด่นของเวอร์ชันนี้

- โฟลเดอร์และโฟลเดอร์ย่อย
- recent reading
- bookmark หลายจุดต่อเล่ม
- จำขนาดตัวอักษรต่อเล่ม
- ออกแบบให้แตะใช้งานง่ายบน iPad Air 5
- รองรับหลายเบราว์เซอร์ผ่าน IndexedDB + fallback
