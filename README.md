# ReadShelf Personal

เว็บแอปอ่านไฟล์ Markdown แบบส่วนตัวสำหรับ iPad โดยตัวโปรแกรมเปิดผ่าน GitHub Pages ได้ ข้อมูลอ่านจะเก็บในเบราว์เซอร์ของเครื่องที่ใช้งานเป็นหลัก และสามารถสำรองขึ้น GitHub Cloud เพื่อเปิดจากอุปกรณ์อื่นได้

## แนวคิดของเวอร์ชันนี้

- GitHub เก็บเฉพาะตัวโปรแกรม
- ข้อมูลการอ่านทั้งหมดเก็บในเครื่องด้วย `IndexedDB`
- ถ้าเบราว์เซอร์ไม่รองรับหรือมีปัญหา ระบบจะ fallback ไป `localStorage`
- มีปุ่มสำรองคลังขึ้น GitHub Cloud เป็นไฟล์แบ็กอัป
- เหมาะกับการเปิดจาก Safari บน iPad แล้วกด `Add to Home Screen`

## เริ่มใช้งานในเครื่อง

```bash
npm install
npm run dev
```

ถ้าต้องการทดสอบจากอุปกรณ์อื่นในวงเดียวกันชั่วคราว:

```bash
npm run dev -- --host
```

## Deploy ขึ้น GitHub Pages

โปรเจกต์นี้ deploy อัตโนมัติเมื่อ push ขึ้น branch `main`

1. push โค้ดขึ้น GitHub
2. ไปที่ `Settings > Pages`
3. เลือก `Source = GitHub Actions`
4. รอ workflow ทำงานเสร็จ
5. เปิดลิงก์ GitHub Pages บน iPad

ตัวอย่าง URL:

```text
https://YOUR-USERNAME.github.io/YOUR-REPOSITORY/
```

## วิธีใช้งานจริง

1. เปิดเว็บแอปบน iPad
2. กด `เพิ่มหนังสือ`
3. เลือกไฟล์ `.md`, `.markdown` หรือ `.txt` จากเครื่อง
4. ถ้าต้องการจัดหมวด ให้กด `สร้างโฟลเดอร์` ก่อนแล้วค่อย import
5. เปิดอ่าน ระบบจะจำ progress และ bookmark ให้อัตโนมัติในเครื่องนั้น

## ความเป็นส่วนตัว

- คนอื่นที่เปิดลิงก์เดียวกันจะไม่เห็นหนังสือส่วนตัวของคุณ
- ถ้าคุณไม่ import ไฟล์เข้าบนเครื่องของเขา เครื่องของเขาจะไม่มีข้อมูลของคุณ
- อย่านำไฟล์ส่วนตัวไปไว้ใน `public/docs` หรือ commit ลง repo ถ้าไม่ต้องการให้เป็นสาธารณะ
- ถ้าใช้ GitHub Cloud ข้อมูลจะถูกเก็บใน repo ตามสิทธิ์ที่กำหนด (public/private)

## การสำรองข้อมูล

แอปมีปุ่ม `แบ็กอัป`, `กู้คืน`, และ `GitHub Cloud`

- `GitHub Cloud` จะอัปโหลดไฟล์แบ็กอัปไปยัง repo ที่ระบุ (ต้องมี Personal Access Token)
- เปิดจากเครื่องอื่นแล้วกด `ดึงแบ็กอัป` เพื่อให้เห็นข้อมูลเดียวกัน

ควรสำรองข้อมูลก่อน:

- ล้างข้อมูลเบราว์เซอร์
- เปลี่ยนเครื่อง
- ย้ายจาก Safari ไปเบราว์เซอร์อื่น
- ทดลองรีเซ็ตแอป

## จุดเด่นของเวอร์ชันนี้

- โฟลเดอร์และโฟลเดอร์ย่อย
- recent reading
- bookmark หลายจุดต่อเล่ม
- จำขนาดตัวอักษรต่อเล่ม
- ออกแบบให้แตะใช้งานง่ายบน iPad Air 5
- รองรับหลายเบราว์เซอร์ผ่าน IndexedDB + fallback
